# QA Analysis & Rules Report - Phase 1 Bugs
**Author:** Maker (@Maker)
**Date:** 2026-06-12

## 개요
본 문서는 서브 에이전트(@sub)의 스트레스 테스트 과정에서 분석된 실패 원인을 바탕으로, 향후 재발 방지를 위해 제정된 개발 및 디버깅 규칙(Rules)을 문서화한 것입니다. 디렉터(@Main)의 지시에 따라 프론트엔드 및 백엔드 개발 시 반드시 준수되어야 합니다.

---

## 1. 프론트엔드 상태 동기화 누락 규칙 (UI Rendering)
**상황:** 오디오 파일 추가 시 백엔드 설정 저장 및 사운드 프리로드가 모두 끝날 때까지 Riverpod의 state가 갱신되지 않아 UI가 멈추는 현상(이슈 1번) 발생.
**근본 원인:** Riverpod 상태 관리가 비동기 작업 종료 후에만 `state`를 덮어쓰도록(Conservative Update) 구현됨.
**해결 및 제정 규칙 (Rule #1): 낙관적 UI 업데이트(Optimistic UI Update) 의무화**
- **규칙:** 사용자의 액션(오디오 추가, 룸 추가 등)으로 인해 로컬 파일 IO 또는 백엔드(Rust FFI)와 통신이 발생하는 경우, **통신 시작 직전에 `state = newConfig` 처리를 선행**하여 UI를 즉각적으로 렌더링해야 한다.
- **적용:** 백엔드 작업 실패 시에는 `try-catch`에서 다시 상태를 롤백하거나 `globalErrorProvider`로 에러 메시지를 띄우는 방식으로 사용자 경험(UX)을 유지할 것.

---

## 2. 하드웨어-소프트웨어 UI 일치화 규칙 (Device Outputs)
**상황:** 환경 설정 내 오디오 아웃풋 라우팅 메뉴가 텍스트 입력창으로 하드코딩되어 있고, 사용자가 요구한 다이얼로그 최하단의 통신 포트 UI 배치가 지켜지지 않음(이슈 2번).
**근본 원인:** 백엔드에서 받아온 `channelNames`를 프론트엔드 UI 컴포넌트로 매핑하는 데이터 브릿지(DropdownItem) 로직 부재.
**해결 및 제정 규칙 (Rule #2): 하드웨어 종속 데이터의 동적 렌더링 원칙**
- **규칙:** CoreAudio/WASAPI에서 `Rust`를 통해 추출된 하드웨어 의존 데이터(예: Analog 1, ADAT 3 등)는 프론트엔드에서 하드코딩된 UI로 처리해서는 안 된다.
- **적용:** 무조건 `DropdownButtonFormField` 등의 선택형 동적 리스트(List Map) 컴포넌트로 렌더링하여 사용자 오입력을 원천 차단하고, UI/UX 기획서의 배치(레이아웃) 요건을 Pixel-perfect 수준으로 준수할 것.

---

## 3. 오디오 디코딩 지원 누락 규칙 (MP3 Parsing)
**상황:** WAV 파일은 정상 재생되나, MP3 파일은 백엔드에서 Decode Fail 발생 (이슈 3번).
**근본 원인:** Symphonia 라이브러리에서 MP3와 같이 명확한 파일 헤더 시그니처가 없는 포맷을 Probe(탐색)할 때 파일 확장자 Hint를 제공하지 않음.
**해결 및 제정 규칙 (Rule #3): 범용 오디오 디코딩 Hint 강제화**
- **규칙:** 백엔드(Rust)에서 `symphonia::core::probe`를 사용할 때, 단순히 파일 스트림만 넘기지 말고 반드시 파일 경로에서 확장자(`extension()`)를 추출하여 `Hint::with_extension`을 함께 넘겨야 한다.
- **적용:** 사용자가 업로드하는 모든 외부 미디어 파일의 디코딩 시, 파일 포맷 유추 실패율을 0%로 만들기 위한 필수 규정.
