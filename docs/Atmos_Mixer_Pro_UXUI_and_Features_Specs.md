# Atmos Mixer Pro - 시스템 기능 및 UX/UI 상세 명세서

**문서 기준 코드**: `main.py`
**목적**: 방탈출 공간 음향 솔루션 'Atmos Mixer Pro' (Phase 2)의 전체 아키텍처, 오디오 엔진 특성, OSC 통신 구조, 그리고 UX/UI의 아주 세밀한 버튼 레이아웃과 동작 방식까지 처음 보는 사람도 완벽히 이해할 수 있도록 상세히 문서화.

---

## 1. 프로그램 목적 및 개요 (Purpose & Overview)

**Atmos Mixer Pro**는 프리미엄 방탈출 테마에서 여러 개의 방(Room)에 설치된 다중 스피커(최대 24채널 하드웨어 랙 시스템, RME + Behringer 등)를 단일 중앙 컴퓨터에서 제어하기 위해 개발된 **시퀀스 제어형 멀티채널 오디오 믹서**입니다.

기존의 단순 재생기를 넘어 다음과 같은 핵심 목적을 가집니다:
1. **아두이노 센서 연동 (OSC 통신)**: 플레이어의 행동(예: 서랍 열기, 버튼 누르기 등)을 아두이노가 감지하여 로컬 네트워크(LAN)를 통해 OSC 신호로 보내면, 해당 신호를 즉시 해석하여 특정 스피커로 오디오를 재생합니다.
2. **독점적 룸 제어 (Exclusive Gating)**: 플레이어가 현재 위치한 방의 오디오 센서 신호만 허용하고, 이전 방이나 다음 방에서 발생하는 센서 오류 신호는 물리적으로 완벽히 차단하여 오디오 간섭을 막습니다.
3. **전문적 오디오 라우팅**: 에이블톤 라이브(Ableton Live)와 같이 특정 오디오 파일을 원하는 오디오 인터페이스의 특정 출력 포트(예: Output 1, Output 3 등)로 직접 라우팅할 수 있습니다.

## 2. 코어 오디오 엔진 (Audio Engine) 및 백엔드 기능

파이썬의 `sounddevice`와 `soundfile`, `numpy`를 활용하여 실시간 고성능 오디오를 처리합니다.

### 2.1. 철저한 스레드 분리 (Thread Separation)
UI가 멈추거나 렉이 걸려도 오디오가 끊기지 않도록 **"스레드 철의 장막"** 원칙을 고수합니다.
*   **메인 UI 스레드**: 화면 렌더링 및 클릭 이벤트만 처리.
*   **오디오 콜백 스레드**: `sounddevice`의 C 레벨 콜백에서 구동되며, 오로지 `numpy` 배열 연산으로 소리를 믹싱(Mixing)하는 역할만 수행.
*   **OSC 수신 데몬 스레드**: 네트워크 포트를 열고 아두이노에서 오는 UDP 패킷만 0.1ms 이내로 수신 대기.

### 2.2. 스마트 오디오 믹싱 효과
*   **소프트 페이드 (Soft Fade In/Out)**:
    *   오디오가 켜지고 꺼질 때 스피커에서 '퍽' 하는 팝 노이즈를 방지하기 위해 0.3초(300ms) 동안 볼륨이 부드럽게 오르내리도록 엔진 단에서 무조건 적용됩니다.
*   **스마트 더킹 (Smart Ducking)**:
    *   효과음(단발성 SFX)이 재생되는 순간, 해당 순간의 몰입도를 높이기 위해 재생 중이던 배경음(BGM)의 볼륨이 0.15초에 걸쳐 **30% 수준으로 자동 감소(Ducking)**합니다.
    *   효과음이 완전히 종료되면 0.3초에 걸쳐 다시 배경음이 **원래 볼륨(100%)으로 부드럽게 복구(Unduck)**됩니다.

## 3. 네트워크 및 룸 배타적 제어 (OSC & Gating)

### 3.1. OSC 통신 및 검증
*   **통신 방식**: UDP 프로토콜을 사용하며 설정된 포트(기본 8000)를 수신 대기합니다.
*   **바운싱 방지 (Debounce)**: 아두이노 물리 센서의 기계적 진동으로 인해 수 밀리초 만에 수십 번의 신호가 오더라도, **250ms 이내에 들어온 동일한 OSC 주소는 무시**하여 오작동을 막습니다.
*   **릴리스 신호 무시**: 센서에서 스위치가 떨어질 때 들어오는 `0` 이하의 값(Release)은 오디오 재생을 방해하지 않도록 무시합니다.

### 3.2. 독점 방 제어 (Exclusive Room Gate)
방탈출은 순차적으로 방을 이동하는 게임입니다.
*   **활성 룸 (Active Room)**: 현재 플레이어가 위치한 방입니다. 해당 방에 속한 오디오 트랙의 재생 신호나 룸 클리어 신호만 허용됩니다.
*   **잠금 룸 (Locked Room)**: 활성화되지 않은 나머지 모든 방은 `Locked` 상태가 되며, UI에서도 컨트롤이 회색으로 비활성화되고 해당 방에서 OSC 신호가 와도 엔진에서 완벽히 **차단(Drop)**합니다.
*   **인터록 (Interlock)**: `룸 1`에서 **"룸 클리어 신호"**를 받으면 진행 상황이 즉시 `룸 2`로 넘어가고, 룸 2의 배경음(BGM)이 자동으로 재생을 시작합니다.

## 4. UI/UX 디자인 및 레이아웃 (세부 스펙)

프로그램은 어두운 환경(테마 통제실 등)에서 엔지니어가 눈부심 없이 볼 수 있도록 다크 블루와 네이비 베이스(`fg_color: #08080F`, `#0E0E1C`)의 **다크 모드 디자인**을 적용했습니다.

### 4.1. 상단 글로벌 헤더 (Global Header)
화면 최상단 영역(높이 90px, 배경색 `#0E0E1C`)으로 메인 컨트롤 스위치들이 존재합니다.

*   **좌측 타이틀**: `🎛 Atmos Mixer Pro` (하얀색), 부제 `시퀀스 제어형 멀티채널 오디오 믹서`
*   **글로벌 제어 버튼 블록 (가운데 좌측)**:
    *   **테마 시작** (파란색 `#1565C0` -> Hover `#0D3E8A`): 모든 룸을 초기화하고 **룸 1**을 독점 활성화하며, 룸 1에 지정된 BGM(무한 루프) 오디오를 즉시 재생시킵니다.
    *   **비상 정지** (빨간색 `#B71C1C` -> Hover `#7F0000`): 긴급 상황(화재, 시스템 오류 등) 발생 시 **모든 오디오를 즉시 뮤트하고 정지(Panic)** 시킵니다.
    *   **시스템 리셋** (어두운 회색 `#37474F` -> Hover `#1C2A30`): 오디오 메모리를 비우고 룸 1번 독점 활성화 모드로 되돌립니다.
    *   **➕ 룸 추가** (초록색 `#2E7D32` -> Hover `#1B5E20`): 새로운 룸(방) 패널을 화면에 하나 추가합니다.
    *   **⚙️ 환경설정** (회색 `#455A64` -> Hover `#263238`): 아웃풋 라우팅과 OSC 신호를 매핑할 수 있는 팝업창을 엽니다.
*   **설정 파일 블록**:
    *   **💾 설정보내기** (보라색 `#6A1B9A` -> Hover `#4A148C`): 현재 UI에 있는 모든 오디오 세팅(볼륨, 경로, OSC 매핑)을 `JSON` 파일로 내보냅니다.
    *   **📂 설정 불러오기** (청록색 `#00838F` -> Hover `#006064`): `JSON` 설정 파일을 선택하여 대시보드를 재건축(Rebuild)합니다.
*   **우측 디바이스 설정 블록**:
    *   `출력 오디오 기기` 라벨 아래에 현재 연결된 오디오 인터페이스(ASIO/Core Audio)를 선택할 수 있는 **드롭다운 메뉴**가 있습니다.
    *   **🔄 스캔** 버튼을 누르면 물리적으로 연결된 스피커 포트(예: Output 1 ~ Output 24)를 하드웨어 레벨에서 다시 스캔합니다.
    *   그 아래에는 현재 연결된 장비의 채널 수를 보여주는 라벨이 표시됩니다.

### 4.2. 메인 화면 - 룸 패널 (Room Panel) 영역
화면 중앙의 거대한 가로 스크롤 영역입니다. 각 방탈출 테마의 방(Room) 단위로 세로로 긴 패널 카드들이 좌에서 우로 나열됩니다.

*   **룸 패널 디자인**: 
    *   크기: 너비 350px, 모서리 10px 라운딩, 배경색 `#0E0E1C`.
    *   순서에 따라 룸 타이틀에 **악센트 컬러** (파랑, 보라, 초록, 핑크 등)가 순환 배정됩니다.
    *   타이틀 우측 끝에는 해당 룸을 통째로 삭제하는 **❌ 버튼** (배경 `#3A1520`)이 위치합니다.
*   **룸 마스터 볼륨**:
    *   우측 상단에 위치한 미니 슬라이더로 해당 방의 전체 오디오 마스터 볼륨 비율을 제어합니다.
*   **버튼 및 잠금 UI**:
    *   **➕ 오디오 파일 추가** (테마 악센트 컬러): 클릭 시 컴퓨터에서 `.wav` 오디오 파일을 불러와 해당 방에 트랙 카드를 추가합니다.
    *   **룸 클리어** (황토색 `#5C4A00`): 클릭 시 강제로 해당 방을 클리어(통과) 처리하고 다음 방으로 넘어갑니다(아두이노 OSC 신호를 마우스로 대리 클릭하는 역할).
    *   현재 활성화된 방이 아닐 경우(예: 현재 룸1 진행 중인데 룸2 화면인 경우) 룸 패널 상단에 **`🔒 잠금 — 이전 룸을 클리어하세요`** 뱃지가 나타나며, 내부의 모든 조작이 불가능(Disabled)해집니다.

### 4.3. 개별 오디오 트랙 카드 (Track Card)
룸 패널 안에 추가되는 각 사운드 소스들의 개별 제어 패널입니다. (배경색 `#161628`)

*   **최상단 우측**: 고정 크기의 붉은색 **X 삭제 버튼** (`#CC0000`). 해당 오디오를 삭제합니다.
*   **중단 첫번째 줄 (제어 & 이름)**:
    *   **재생 버튼 (▶)**: 초록색 (`#1E6B22`). 누르면 수동으로 오디오를 재생하며 아이콘이 정지(⏹, 붉은색 `#8B0000`)로 바뀝니다. OSC 신호 없이 마우스로 강제 테스트할 때 사용합니다.
    *   **이름 엔트리(입력칸)**: 텍스트 박스로, 오디오 파일의 원래 이름을 보여주며 직접 타이핑해 원하는 이름(예: `서랍 열림 소리`)으로 바꿀 수 있습니다.
*   **중단 두번째 줄 (볼륨 조절)**:
    *   **볼륨 슬라이더**: 0~1 (0%~100%)까지 조절 가능한 슬라이더. 룸의 악센트 색상을 따라갑니다. 우측에 퍼센트 텍스트가 표시됩니다.
*   **하단 줄 (루프 및 라우팅)**:
    *   **🔄 무한 루프 스위치**: 이것을 켜면 해당 트랙은 단발성 효과음(SFX)이 아닌 **배경음(BGM)**으로 취급됩니다. 룸에 입장하면 자동으로 루프(반복) 재생됩니다.
    *   **🔌 Output 채널 라벨**: 우측 끝에 이 소리가 어느 하드웨어 스피커 채널(예: Output 3)로 출력될 지 텍스트로 표시해 줍니다.

### 4.4. 환경설정 팝업 다이얼로그 (Settings Dialog)
헤더의 ⚙️ 버튼을 누르면 뜨는 독립된 설정 창(720x560 크기)입니다.

*   **오디오 아웃풋 탭**:
    *   룸에 등록된 모든 오디오 트랙들이 목록으로 나타나며, 우측에 드롭다운 메뉴가 나타납니다.
    *   여기서 해당 소리가 실제 오디오 인터페이스의 **몇 번 채널로 나갈지 (예: 1: Output 1, 2: Output 2)** 하나하나 물리적 라우팅을 매핑할 수 있습니다.
*   **아두이노 신호 탭 (OSC 매핑)**:
    *   **룸 클리어 아두이노 신호**: 해당 방을 깼다는 아두이노의 센서 신호 주소(예: `/room1/clear`)를 입력합니다.
    *   **트랙별 재생/정지 신호**: 각 트랙카드 별로 특정 센서가 반응했을 때 낼 재생 주소(예: `/room1/drawer/play`)와 정지 주소를 타이핑하여 매핑합니다.
*   **OSC 수신 포트 설정**: 다이얼로그 최하단에 통신을 받을 포트 번호(기본 8000)를 입력할 수 있습니다.
*   **저장 및 닫기 버튼**: 파란색(`#1565C0`), 클릭 즉시 정보가 엔진에 동기화되고 `config.json`에 기록되며 팝업이 닫힙니다. 무거운 오디오 재연결 작업은 스레드 락을 피하기 위해 백그라운드에서 안전하게 수행됩니다.

### 4.5. 하단 시스템 로그 (System Log)
윈도우 맨 아래 위치한 개발자용 실시간 모니터링 콘솔입니다.

*   **크기 및 색상**: 높이 155px 고정, 순수 검정(`#000000`) 바탕에 밝은 초록 글씨(`#22DD88`)를 사용해 해커/서버 터미널 감성을 줍니다.
*   **출력 내용**: 오디오 포트 인식 성공, OSC 신호 수신(몇 밀리초 걸렸는지), 트랙 재생/정지, 룸 이동, 설정 저장 완료, 그리고 에러 메시지(경고) 등 시스템의 모든 상태를 타임스탬프와 함께 출력합니다.
*   **우측 상단 [지우기] 버튼**: 클릭 시 그동안 쌓인 로그 텍스트를 깨끗하게 비웁니다.

### 4.6. 글로벌 부드러운 스크롤 (Advanced Touchpad/Mouse Wheel Support)
파이썬 `tkinter` 특유의 끊기는 마우스 휠 스크롤 경험을 극복하기 위해 네이티브 레벨의 스크롤 패치(`AtmosScrollableFrame`)를 적용했습니다. 맥북(Mac) 트랙패드나 윈도우 정밀 터치패드, 마우스 휠로 어느 곳을 긁더라도 가로 룸 패널과 세로 트랙 카드, 환경설정 리스트 등이 아주 매끄럽고 부드럽게 스크롤됩니다.

---

Atmos Mixer Pro는 잦은 오작동이 일어날 수 있는 오프라인 센서 환경을 소프트웨어의 강력한 유효성 검증과 "독점적 게이팅" 구조로 방어하고, 직관적이면서도 안전성을 최우선으로 고려한 전문가용 인터페이스로 완성된 작품입니다.
