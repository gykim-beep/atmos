# ♾️ Loop System (Autonomous Supervisor)

이 문서는 코더 에이전트(`@coder`)가 작업을 수행하고 스스로 결과를 검증하기 위한 **무한 검증 루프(Infinite Verification Loop)** 시스템입니다.
코딩 완료를 선언하기 위해서는 아래의 9가지 필수 통과 기준을 **모두 100점 만점으로 통과**해야 합니다. 단 하나라도 통과하지 못하면, 수정 후 다시 루프를 돌아야 합니다.

---

## 🛑 필수 통과 기준 (나노 단위 검증 체크리스트)

### 1. PRD를 확인했는가? (100점)
- [ ] [10점] PRD의 핵심 목표(Zero-Allocation, Lock-Free Audio 등)를 침해하는 로직이 없는가?
- [ ] [30점] 오디오 콜백 루프 내에서 String 생성, Vec 힙 메모리 동적 할당 원천 차단 확인.
- [ ] [30점] 오디오 콜백 내 디스크 파일 읽기/쓰기(`File::open`) 로직 원천 차단 확인.
- [ ] [30점] 사용자가 요구한 모든 기능 명세가 누락 없이 구현되었는가?

### 2. 유저플로우와 맞는가? (100점)
- [ ] [30점] 시스템 리셋 -> 테마 시작 버튼 클릭 시 UI 상태 불일치 버그가 발생하지 않는가?
- [ ] [30점] OSC 트리거 및 UI 조작 시 의도한 순서대로 동작하며, 멈춤이나 데드락이 없는가?
- [ ] [40점] 오류 발생 시 사용자에게 명확하게 에러 배너(UI)로 피드백을 전달하는가?

### 3. Screens에 반영됐는가? (100점)
- [ ] [40점] 화면 리사이징 시 UI가 깨지거나 "RIGHT OVERFLOWED BY XXX PIXELS" 경고가 뜨지 않는가?
- [ ] [30점] 각 컴포넌트(`DashboardScreen`, `RoomCard` 등)의 상태 관리가 Riverpod를 통해 정상적으로 이루어지고 있는가?
- [ ] [30점] 화면 전환이나 테마 변경 시 이전 상태의 잔재가 스크린에 남아있지 않은가?

### 4. DB 설계와 충돌하지 않는가? (100점)
- [ ] [50점] `config.json` 저장 및 불러오기 시 직렬화/역직렬화 오류가 없는가?
- [ ] [50점] Rust의 `AppConfig` 구조체와 Dart의 모델 구조가 1:1로 정확하게 일치하는가?

### 5. 디자인 시스템을 지켰는가? (100점)
- [ ] [50점] 프로젝트에 정의된 `AppColors` (예: 네온 포인트, 다크 그레이 등) 및 폰트 스타일을 위반한 하드코딩된 색상이 없는가?
- [ ] [50점] 버튼 호버 및 클릭 시 애니메이션(Glow 효과 등)이 기존 디자인 원칙과 부합하는가?

### 6. Tasks에 언급된 사항을 빠뜨리지 않았는가? (100점)
- [ ] [50점] 현재 진행 중인 Task의 세부 요구사항 목록을 100% 충족했는가?
- [ ] [50점] TODO 주석이나 임시로 막아둔(`unimplemented!()`) 코드가 프로덕션 코드에 남아있지 않은가?

### 7. 코딩 컨벤션스를 따랐는가? (100점)
- [ ] [40점] Rust: 문서화(100% 한글), 에러 핸들링, `unwrap()` 남용 방지 원칙을 지켰는가?
- [ ] [30점] Dart: Riverpod 상태 관리 패턴 및 파일 분리 원칙을 준수했는가?
- [ ] [30점] 네이밍 규칙(카멜, 스네이크, 파스칼 케이스)이 기존 코드베이스와 일치하는가?

### 8. 테스트, 타입체크, 린트, 빌드를 통과했는가? (100점)
- [ ] [20점] Rust `cargo check` 및 `cargo clippy` 경고 0건인가?
- [ ] [20점] Dart `flutter analyze` 경고 0건인가?
- [ ] [30점] macOS 환경에서 `flutter build macos`가 에러 없이 성공하는가?
- [ ] [30점] **객관적 증거 제출**: 빌드 성공 로그, 린트 통과 로그, 레이스 컨디션 테스트(예: stress_test) 결과를 제시할 수 있는가?

### 9. 남은 리스크는 무엇인가? (100점)
- [ ] [50점] 메모리 누수, CPU 점유율 스파이크, 또는 동시성 문제(Race Condition)에 대한 방어 로직이 완벽한가?
- [ ] [50점] 보안 키(보안 코드 등) 노출 여부를 점검하였으며 이상이 없는가?

---

## 🛠️ Loop 동작 방식
1. `@coder`는 구현을 마친 후 위 9가지 체크리스트(총 900점)를 스스로 평가해야 합니다.
2. 각 항목별로 **통과한 이유와 객관적 증거**를 제시해야 합니다.
3. 9가지 기준 중 하나라도 100점이 안 되는 항목이 있다면, 코딩 완료를 선언하지 말고 즉시 코드를 수정해야 합니다.
4. 모든 기준을 100점으로 통과했을 때 비로소 **"Loop Verification Passed"**를 선언하고 사용자에게 리뷰를 요청합니다.
